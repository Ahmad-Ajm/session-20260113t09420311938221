# Plan – FEAT-UX-BASE

## Goal
بناء واجهة مستخدم أساسية (Static UI) للمنصة بدون أي تفاعل أو منطق أعمال، بهدف توفير أساس بصري موحّد يمكن البناء عليه لاحقًا عند إضافة الميزات التفاعلية.

## In Scope
- تصميم Layout عام للمنصة:
  - Header (شعار + اسم المنصة)
  - Content Area (أقسام الصفحة)
  - Footer (نص ثابت)
- صفحة واحدة تحتوي على أقسام ثابتة (Placeholders):
  - قسم الاسم
  - قسم رقم الجوال
  - قسم الصورة الشخصية
  - قسم الخبرات
  - قسم التعليم
- استخدام عناصر UI وهمية (Mock Cards / Static Blocks).
- دعم اتجاه RTL بشكل كامل.
- HTML / CSS فقط (أو JSX بدون منطق إذا استُخدم React).

## Out of Scope
- أي تفاعل (Click, Submit, Validation).
- أي منطق Backend أو APIs.
- ربط البيانات أو تخزينها.
- Responsive متقدم (يُكتفى بتخطيط بسيط قابل للعرض).

## Assumptions
- استخدام Framework Frontend حديث (React أو HTML/CSS بسيط).
- جميع البيانات المعروضة هي Placeholder فقط.
- لا يوجد تكامل مع ميزات أخرى في هذه المرحلة.

## Milestones
1. إعداد الهيكل العام للصفحة (Layout Skeleton).
2. بناء الأقسام الداخلية بشكل ثابت.
3. تطبيق تنسيق بسيط ومحايد يدعم العربية.

## Acceptance Criteria
- الصفحة تُعرض بدون أخطاء.
- جميع الأقسام المطلوبة موجودة كعناصر ثابتة.
- الواجهة تدعم RTL.
- لا يوجد أي تفاعل أو منطق برمجي.
